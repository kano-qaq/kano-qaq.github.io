(RE01~RE03)
签名文件制作:
1.link -lib /list 目标.lib 显示目标库中的所有obj
2.link -lib /extract:object1.obj 目标.lib 提取出其中一个obj
3.pcf object1.obj 将.obj文件转为.pat文件
4.sigmake -n”注释” object1.pat object1.sig 制作签名文件

只有单操作数乘法扩展位长。
除法只有除数不为变量时可以优化。

无符号作被除数:
除数为2^n时:
除数大于0:shr 被除数,n
除数小于0:div 除数

除数不为2^n时:
除数的10进制表达式=floor(2^(32+右移位数)/magic_number)
将除数的10进制表达式转为16进制判断正负
(乘减移加移的真实magic_number = 1 0000 0000h + 反汇编显示的magic_number)

vc6中:
unsigned int / -5 和 unsigned int / 4294967291 汇编代码一样
vs2019中:
表达式unsigned int / 4294967291的4294967291会默认转为long long类型，导致引入大数乘法。
unsigned int / -5 和unsigned int / 4294967291i32 的汇编代码一样

有符号作被除数:
除数为2^n时:

除数不为2^n时:
magic_number<0,没有任何调整代码=>除数为负(magic_number为求补后的)
magic_number<0,有加调整=>除数为正
magic_number>0,有减调整=>除数为负(magic_number为求补后的)
magic_number > 0,没有任何调整代码=>除数为正
