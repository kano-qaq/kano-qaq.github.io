SQL语法
单表查询:
select column_name1,… from table_name
从table_name表中查询column_name1表项的值

别名:
select column_name1 as new_name,… from table_name
从table_name表中查询column_name1表项的值，但是显示结果时column_name1变为new_name

字段运算:
select column_name1-10 from table_name
从table_name表中查询column_name1表项的值，减去10的结果

where表达式:
select column_name1 from table_name where expr
从table_name表中查询符合expr表达式的column_name1表项的值

集合:
select * from table_name where column_name1 in (xxx,yyy)
从table_name表中查询column_name1表项在集合(xxx,yyy)中的所有值

范围:
select * from table_name where column_name1 between x and y
从table_name表中查询column_name1表项的值大于等于x且小于等于y的所有值

排序:
select * from table_name order by column_name1 [asc/desc]
从table_name表中查询按column_name1[升序/降序]的所有值

模糊匹配
select * from table_name where column_name1 like ‘a% / _b% / a_b / %a%’
匹配a开头的字符串/任意字符开头第二个字符为b后接任意字符的串/
第一个字符为a第二个字符任意第三个字符为b的串/包含字符a的任意串

多表查询
select * from table_name1, table_name2
将从table_name1和table_name2查询到的结果拼成一张表

自然连接
select * from table_name1 natural join table_name2

并集:
(集合1) union (集合2)

交集:
(集合1) intersect (集合2)

并集
(集合1) except (集合2)

子查询
select * from table_name1 where column_name1 =
(select column_name1 from table_name2 where column_name2 = xxx)

测试子查询
select * from table_name1 where exists
(select * from table_name2 where table_name1.column_name1 = table_name2.column_name1)
exists关键字可以提升查询效率

指定结果显示个数和从哪开始显示:
select * from table_name order by column_name desc limit show_count offset start_offset
select * from table_name order by column_name desc limit start_offset, show_count

聚集函数:
求平均值avg(),求总和sum(),求最大max(),求最小min(),求个数count()

分组(需要和聚集函数搭配使用):
select column_name1,avg(column_name2) from table_name group by column_name1
查询table_name表中由column_name1的值为分组的column_name2的平均值

临时表:
select * from(
select * from table_name
)t1
查询由查询table_name得到的结果作为表的所有表项

having:
为分组添加限制条件
select column_name1,avg(column_name2) from
table_name group by column_name1
having xxx = xxx

内连接:
select * from table_name1 inner join table_name2
on table_name1.column_name = table_name2.column_name

约束(数据完整性)
1、主键约束 primary key 确保字段值不重复不为NULL PK
2、外键约束 foreign key 确保字段值必须来自于指定表 FK
3、唯一约束 unique 确保字段值不重复
4、检查约束 check 确保字段值的取值范围
5、缺省约束 default 给相应字段提供默认值

实体关系转表：
1 对 1 合并两张表
1 对 m 少的主键当做多的外键
m 对 n 创建一个新表（关系表名）， 将两个表的主键当做新表的复合主键

3范式(3NF)
第一范式：必须有主键， 字段原子
第二范式：所有非关键字段都是有主关键字决定
第三范式：所有非关键字段都是有复合关键字共同决定

socket模型
选择模型(轮询设计)
1.在连接时需要手动保存客户端的socket
2.通过函数select完成对每个socket的轮询
3.再通过循环检测有数据的socket

异步选择模型(往窗口发消息)
1.通过函数WSAAsyncSelect()为socket注册事件
2.在接受连接时通过WSAAsyncSelect()为客户端socket注册事件
3.在消息处理函数做处理

异步事件模型(使用事件通知)
1.每连接一个客户端就通过WSACreateEvent()创建一个事件，并且通过WSAEventSelect将事件和客户端socket绑定
2.通过WSAWaitForMultipleEvents()等待有信号事件，返回值为事件数组的下标

重叠模型
1.声明一个特定的结构体数组
结构体:
typedef struct
{
WSAOVERLAPPED overlap;
WSABUF Buffer;
char szMessage[MSGSIZE];
DWORD NumberOfBytesRecvd;
DWORD Flags;
}PER_IO_OPERATION_DATA
2.在接受连接时初始化结构体的信息
3.调用WSARecv()声明如果有数据就放到结构体指定的缓冲区中
4.通过WSAWaitForMultipleEvents()等待有信号事件，返回值为事件数组的下标
5.通过WSAGetOverlappedResult()获取是否有数据的结果

完成端口模型(线程池+重叠模型)

http协议

1.版本:
http1.0使用tcp连接
http1.1使用tcp连接,默认端口为80
http2.0使用tcp连接,默认端口为443(https)
http3.0使用可靠udp连接

2.常用方法:
GET,POST,HEAD

3.ssl原理
客户端与服务端建立连接时，服务端向客户端发送一个公钥
客户端使用公钥加密对称加密的Key
服务端使用私钥解密Key，再用Key解密数据

4.双向验证
客户端与服务端建立连接时，客户端会对服务端进行验证，而且服务端也会对客户端进行验证

5.中间人攻击
客户端连接服务端时连接到伪造的服务端，与伪造服务端进行ssl商议。
伪造服务端再与真实的服务端进行ssl商议。
伪造服务端用自己的私钥解密客户端数据再用真实客户端的公钥加密发给服务端
