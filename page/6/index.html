note

virtual 虚函数

覆盖：
1.必须使用基类的指针或者引用调用虚函数
2.派生类函数与基类同名，同参，同返回值
3.基类的函数必须由virtual关键字
4.如果基类由virtual关键字，派生类默认有

隐藏：

注意：
构造函数不能多态
构造函数调用虚函数不多态
析构函数调用虚函数不多态

override关键字检测派生类是否重写了基类函数。
final关键字用于禁止后续同名函数覆盖。

析构函数允许不同名的多态。
显式调用析构函数会修改虚函数表。

Posted on 2023-04-27 Edited on 2023-05-18

note

拷贝构造的触发：
1.函数返回的类型为对象
2.传入的参数为对象

移动构造形式
Classname(Classname&& obj){…}
移动构造的触发：
1.对象作为返回值且没有使用
2.无名对象作为参数
3.强转为右值引用
4.std::move(对象标识符)

友元
1.友元全局函数 friend int main();
2.友元成员函数 friend void Classname::function();
3.友元类 friend class Classname;

运算符重载
目的：通过运算符来调用成员函数
注意：
1.不改变运算符的原义
2.运算符优先级不变
3.使用方式不变

成员函数重载，要求符号的左侧必须为对象。
全局函数运算符重载，可以实现符号的左侧为非对象。但是需要友元联系使用。

转换运算符形式
operator return_type {…}

转换优先级低于重载

内联关键字 inline

lambda

组成部分有：
1.捕获模式

  • 1.1[]表示在主体不访问任何变量。[=]可以访问变量的值。[&]变量的引用访问。
  • 1.2若一个lambda表达式引用访问变量t和访问f的值，以下等效[&t,f];[f,&t];[&,f];[=,t];
  • 1.3在捕获模式中若已存在&/=，那么后面的就不可以再次出现。
  • 1.4标识符和this只能出现一次
